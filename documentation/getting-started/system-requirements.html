Purpose

Use this page to confirm that a device, browser, display, network, and security environment are suitable for True Care System.

Supported Access Methods

True Care System can be accessed through:

  • A supported desktop or laptop web browser.
  • The True Care System mobile application for supported EVV workflows.
  • Organization-approved devices managed according to internal security policy.

The features available on a device depend on the user's assigned role, permissions, and the capabilities of that device.

Recommended Web Environment

Component Recommendation
Browser Current version of Microsoft Edge or Google Chrome.
Operating System A currently supported Windows or macOS version.
Screen Resolution 1920 × 1080 recommended; lower resolutions may require additional scrolling.
Browser Zoom 100% recommended for standard layout and documentation screenshots.
Internet Stable broadband connection with reliable upload and download performance.
PDF Support Browser PDF viewer or organization-approved PDF reader.
Pop-ups and Downloads Allow downloads and approved pop-ups required for reports, exports, and print views.

Browser Configuration

For reliable operation, the browser should allow:

  • JavaScript.
  • Secure cookies required for authentication and session management.
  • File downloads for approved reports and exports.
  • Printing and PDF generation when required.
  • Access to location services when a permitted workflow requires it.

Browser extensions that block scripts, cookies, downloads, or authentication requests may prevent some features from working.

Mobile EVV Requirements

Mobile EVV workflows may require additional device capabilities.

  • A supported Android phone or iPhone.
  • The current approved version of the True Care System mobile app.
  • Location Services enabled while completing EVV actions.
  • Permission for the mobile app to use GPS.
  • A reliable mobile data or Wi-Fi connection.
  • Accurate device date, time, and time zone settings.
  • Sufficient battery power for the complete visit.
GPS requirement

Disabling location access may prevent check-in, check-out, or other EVV functions from completing successfully.

Network Requirements

The organization should provide:

  • A stable internet connection.
  • HTTPS access to True Care System domains.
  • Access to approved authentication, API, storage, and integration endpoints.
  • Firewall and web-filter rules that do not block required application traffic.
  • Reliable DNS resolution.

Network interruptions may affect saves, uploads, report generation, EVV activity, or integration submissions.

Device Security Requirements

Devices used to access True Care System should:

  • Require a password, PIN, biometric control, or approved authentication method.
  • Use current operating-system and browser security updates.
  • Use organization-approved endpoint protection when required.
  • Automatically lock after a period of inactivity.
  • Be protected from unauthorized physical access.
  • Not be rooted, jailbroken, or otherwise security-compromised.

HIPAA Considerations

The device and environment used to access True Care System may display Protected Health Information. Organizations should apply appropriate administrative, physical, and technical safeguards.

  • Do not use public or shared computers for PHI unless specifically approved.
  • Prevent unauthorized persons from viewing the screen.
  • Do not download PHI to unmanaged devices.
  • Do not store screenshots containing real PHI in public repositories.
  • Use secure networks and avoid untrusted public Wi-Fi.
  • Sign out and secure the device when work is complete.

Validation Checklist

Before beginning work, verify that:

  • The login page opens without a browser security warning.
  • The browser is current and supported.
  • The page layout displays correctly at 100% zoom.
  • The device date, time, and time zone are accurate.
  • Reports and downloads can be opened when permitted.
  • Location access is available for approved mobile EVV workflows.
  • The device and network comply with organizational security policy.

Troubleshooting

The page does not load

Confirm internet access, verify the URL, refresh the page, and check whether the organization firewall or web filter blocks the site.

The page layout appears incorrect

Set browser zoom to 100%, maximize the browser window, and confirm that the browser is current.

A report or export does not open

Confirm that downloads and approved pop-ups are allowed. Check the browser download area and the organization's file-security policy.

Mobile check-in cannot verify location

Confirm that device Location Services and app GPS permission are enabled and that the device has a reliable connection.

Frequently Asked Questions

Can I use another browser?

Another modern browser may work, but Microsoft Edge and Google Chrome are the recommended supported browsers unless your organization provides different instructions.

Can I use a tablet?

Some web functions may work on a tablet, but desktop or laptop access is recommended for complex administrative, billing, payroll, reporting, and configuration workflows.

Do I need GPS for all True Care System functions?

No. GPS is primarily required for approved Mobile EVV workflows and any other feature that explicitly requires location verification.

Can I use public Wi-Fi?

Use only networks approved by your organization. Untrusted public Wi-Fi may create security and privacy risks.
