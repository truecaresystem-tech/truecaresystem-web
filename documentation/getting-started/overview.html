This page is intended for new True Care System users, Provider Administrators, office staff, supervisors, DSPs, billing staff, payroll staff, compliance staff, and system trainers.

Purpose

The Getting Started guide introduces True Care System and explains the basic information users need before working with the platform.

Use this guide to understand the purpose of the system, the primary modules, account requirements, recommended security practices, and the suggested learning path for new users.

About True Care System

True Care System is a care management platform designed for organizations that provide home care, intellectual and developmental disability services, community-based services, and other person-centered support programs.

The platform brings administrative, workforce, scheduling, EVV, visit management, billing, payroll, reporting, security, and compliance workflows into one coordinated system.

True Care System is designed to help providers:

  • Manage employees and individuals in one secure platform.
  • Create schedules and assign qualified staff.
  • Capture Electronic Visit Verification information.
  • Review and maintain completed visits.
  • Prepare billing information and Sandata submissions.
  • Support payroll processing and operational reporting.
  • Maintain security controls and audit records.
  • Support HIPAA-compliant operational practices.

Who Should Use This Documentation

This documentation supports users with different responsibilities throughout a provider organization.

  • Provider Administrators: Configure the organization, users, roles, permissions, services, and system settings.
  • Office Staff: Maintain employee, individual, schedule, visit, and operational information.
  • Supervisors: Review schedules, service delivery, documentation, and employee activities.
  • Direct Support Professionals: View assigned visits and complete EVV-related activities.
  • Billing Staff: Review readiness, visits, claim batches, and Sandata submissions.
  • Payroll Staff: Review employee rates, work time, adjustments, and payroll output.
  • Compliance and Security Staff: Review HIPAA controls, access activity, audit logs, and security events.
  • Developers and QA Staff: Review technical standards, APIs, expected behaviors, and test requirements.

Available pages and actions depend on the user's assigned role and permissions.

Recommended Learning Path

New users should review the Getting Started pages in this order:

  1. Review System Requirements to confirm that the device and browser are ready.
  2. Review Login to learn how to access the system securely.
  3. Review Forgot Password before assistance is needed.
  4. Review Dashboard to understand the initial workspace after login.
  5. Review System Navigation to learn how menus, pages, filters, and actions are organized.
  6. Review User Profile to maintain account and personal settings.
  7. Continue to the module documentation that matches the user's assigned responsibilities.

Before You Begin

Before using True Care System, confirm the following:

  • Your organization has an active True Care System account.
  • Your user account has been created by an authorized administrator.
  • Your assigned role matches your job responsibilities.
  • You have received the correct login address.
  • You are using a supported browser and a secure internet connection.
  • You understand your organization's privacy and security policies.
  • You know how to contact an administrator or support representative.

Security and HIPAA Responsibilities

True Care System may contain Protected Health Information and other confidential organizational information. Every user is responsible for protecting the information available through the account.

Users must:

  • Use only their individually assigned account.
  • Keep passwords and verification codes confidential.
  • Access only information needed for assigned responsibilities.
  • Lock or sign out of unattended devices.
  • Avoid displaying PHI where unauthorized persons can view it.
  • Report suspected unauthorized access or disclosure promptly.
Minimum Necessary Standard

Access only the minimum amount of information required to complete the assigned task.

Audit Logging

True Care System uses audit logging to support accountability, security monitoring, compliance review, and investigation.

Depending on system configuration, audit records may include:

  • Successful and failed login attempts.
  • Logout and session expiration events.
  • Password and account security events.
  • Access to protected records.
  • Creation, update, deletion, and status changes.
  • Support impersonation activity.
  • Billing, payroll, and administrative actions.

Audit records are available only to authorized users with the required permissions.

Best Practices

  • Use the latest supported version of the browser.
  • Do not share user accounts.
  • Review page messages before continuing to another action.
  • Verify information before selecting Save, Submit, Delete, or Approve.
  • Keep employee and individual information accurate and current.
  • Use organization-approved devices and internet connections.
  • Sign out when work is complete.

Frequently Asked Questions

Do all users see the same menus?

No. Menus and available actions depend on the user's role, permissions, provider assignment, and system configuration.

Can one employee share an account with another employee?

No. Each user must use an individually assigned account so that access and activity can be correctly secured and audited.

What should I do if a required module is missing?

Contact an authorized Provider Administrator. The account may not have the required role or permission.

Should I use real information when creating training screenshots?

No. Use demo or training data. Do not publish screenshots containing real PHI, passwords, Social Security numbers, full Medicaid IDs, private notes, or security credentials.
