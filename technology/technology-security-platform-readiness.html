Technical Overview

Security, compliance and SaaS readiness for provider organizations

This page explains how this area of True Care System supports provider operations, EVV readiness, data quality, security, and scalable SaaS delivery.

Security by Design

Security controls are built into the platform architecture rather than added as an afterthought.

  • JWT authentication
  • HTTPS communication
  • Role-based access
  • Secure API design

Provider Isolation

The multi-provider SaaS model separates data by authenticated provider context and limits user visibility by role.

  • Provider-scoped records
  • No frontend-trusted provider switching
  • User role permissions
  • Administrative oversight

Audit and Accountability

Operational changes, visit updates, billing activity and approval history can be tracked to support compliance and review.

  • Visit change history
  • Billing updates
  • Approval history
  • Rejection and resubmission activity

Platform Readiness

The system is built for growth across provider organizations, mobile teams, billing workflows and integration needs.

  • Cloud-ready deployment
  • Scalable architecture
  • Backup-oriented design
  • Enterprise onboarding support
