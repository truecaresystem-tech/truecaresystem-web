Technical Overview

Cloud-native architecture for multi-provider care management

This page explains how this area of True Care System supports provider operations, EVV readiness, data quality, security, and scalable SaaS delivery.

Connected Platform Layers

The architecture organizes operational functions into clear layers from user devices to API services and database storage.

  • Web Portal for administrative workflows
  • Mobile App for field care teams
  • NestJS API layer for business logic
  • PostgreSQL database with Prisma ORM

Provider Isolation by Design

True Care System supports multiple providers while keeping each organization’s data separated and protected.

  • Provider scope is derived from authenticated users
  • Frontend provider IDs are not trusted for data isolation
  • Role-based permissions control visibility
  • Audit logs support accountability

Modern Technology Stack

The platform uses modern web, mobile, backend, ORM, and database technologies suitable for SaaS growth.

  • Next.js + TypeScript for the Web Portal
  • React Native + Expo for iOS and Android
  • NestJS + TypeScript for backend APIs
  • PostgreSQL for structured data

Scalable Operational Foundation

The system is built to support scheduling, EVV, billing, claims, payroll, reports, KPI, and compliance workflows.

  • Cloud deployment model
  • Secure HTTPS communication
  • Operational modules share one data foundation
  • Designed for future integrations
